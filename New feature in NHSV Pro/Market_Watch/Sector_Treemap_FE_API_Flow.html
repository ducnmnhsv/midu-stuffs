2 Step-by-step Flow
1

User mở Market Watch → Sector Treemap render

Trigger: componentDidMount / useEffect([], []) — gọi ngay khi screen mount
GET /api/v1/marketWatch/sectorIndex Dữ liệu index thời gian thực
Gọi song song với API 2 Poll realtime 15–30s

Fields dùng trong UI:

sectorId sectorName perChange → màu ô tradingValue → tab GTGD tradingVolume → tab KLGD foreignBuyVolume → KLNN mua foreignSellVolume → KLNN bán change · close · tradingDate
GET /api/v1/marketWatch/sectorDetail Vốn hóa, PE/PB, lịch sử
Gọi song song với API 1 Cache 5–15 phút (không poll realtime)

Fields dùng trong UI:

sectorId marketCap → tab Vốn hóa pe · pb · eps perChange1W · 1M · 3M · 6M · 52W high52W · low52W

JOIN STRATEGY sau khi cả 2 API trả về:

sectorIndex.list ~15 sectors · perChange · tradingValue · tradingVolume · …
+
sectorDetail.list keyed by sectorId · marketCap · pe · pb · perChange1W…
→
cells[] — input cho d3.treemap() Filter: sectorLevel === 1
Merge by: sectorId
Sort: desc by sizeMetric (tab active)
Render strategy: Có thể render treemap ngay khi API 1 về (dùng sectorIndex only), rồi enrich khi API 2 về (enable tab Vốn hóa). Tránh block toàn bộ UI khi sectorDetail chậm.
2

User chọn tab khác (Khối lượng GD / Vốn hóa / …)

Trigger: Tap tab · Không gọi API mới
✓ Không cần gọi API — data đã có trong memory từ Step 1. FE chỉ re-run d3.treemap() với sizeMetric mới và animate layout transition 300ms.
Tab active sizeMetric field Source API Cell color field
Giá trị GD tradingValue API 1 · sectorIndex perChange (API 1)
Khối lượng GD tradingVolume API 1 · sectorIndex perChange (API 1)
Vốn hóa marketCap API 2 · sectorDetail perChange (API 1)
KLNN mua (optional) foreignBuyVolume API 1 · sectorIndex perChange (API 1)
KLNN bán (optional) foreignSellVolume API 1 · sectorIndex perChange (API 1)
⚠ Tab Vốn hóa: Nếu API 2 (sectorDetail) fail → disable tab Vốn hóa, hiển thị tooltip "Không có dữ liệu vốn hóa". Các tab khác vẫn hoạt động bình thường.
3

User tap vào ô ngành → Drill-down Stock Treemap

Trigger: onPress(cell) → navigate sang drill-down view với sectorId + sectorName
GET /api/v1/marketWatch/sectorStocks?sectorId={id} Danh sách mã CK trong ngành
On-demand — gọi khi tap Poll realtime 15–30s khi đang xem

Fields dùng trong UI:

stockCode stockName · exchange perChange → màu ô stock tradingValue → tab GTGD tradingVolume → tab KLGD marketCap → tab Vốn hóa foreignBuyVolume · foreignSellVolume
Backend aggregate flow (FE không cần biết, BE xử lý):
GET /api/v1/marketWatch/sectorStocks?sectorId=X
  ├─ [1] Vietstock /GetListStockBySector?sectorID=X (EOD, cache 24h)
  │        → danh sách stockCode trong ngành
  └─ [2] TradeX /rest/api/v2/market/symbol/latest?codes={stockCodes} (realtime)
          → perChange, tradingValue, tradingVolume, marketCap…
  → BE merge [1]+[2] on stockCode → trả về response
Tab trong drill-down: Kế thừa tab đang active từ sector view. Cùng 1 API call trả đủ data cho tất cả 5 tabs — không cần gọi lại khi switch tab trong drill-down.
4

User chọn tab trong drill-down view

Trigger: Tap tab · Không gọi API mới
✓ Data đã có từ sectorStocks response (Step 3). Re-run d3.treemap() với sizeMetric mới, animate 300ms. Logic giống hệt Step 2 — chỉ khác source data là stocks thay vì sectors.

6

User nhấn Back → về Sector Treemap

Trigger: Back button · Swipe back gesture · Không gọi API mới
✓ Sector data đã cache trong memory. Không re-fetch. Tab state giữ nguyên. Polling sectorIndex tiếp tục nếu đang chạy (không cancel).
7

Pull-to-refresh (bất kỳ view nào)

Trigger: onRefresh handler
Sector view (Screen 1): Gọi lại song song API 1 (sectorIndex) + API 2 (sectorDetail). Reset polling timer.
Drill-down view (Screen 2): Gọi lại API 3 (sectorStocks) với cùng sectorId đang xem. Reset polling timer.
3 Polling Schedule
API 1 — sectorIndex
15 – 30s
Chỉ khi market open (9:00–15:00). Dừng khi user navigate khỏi screen.
API 2 — sectorDetail
Không poll
Cache 5–15 phút ở BE. FE chỉ fetch 1 lần khi mount. Refetch khi pull-to-refresh.
API 3 — sectorStocks
15 – 30s
Chỉ khi đang trong drill-down view. Dừng ngay khi back về sector view.
4 Error States & FE handling
Scenario Severity API bị lỗi FE behavior
sectorIndex fail (timeout / 5xx) HIGH API 1 Empty state toàn bộ treemap · Hiện error banner · Cho phép retry
sectorDetail fail MED API 2 Vẫn render treemap (dùng API 1 only) · Disable tab "Vốn hóa" · Tooltip giải thích
sectorStocks fail (drill-down) MED API 3 Empty state trong drill-down view · Back button vẫn hoạt động
sectorStocks trả empty array LOW API 3 "Chưa có dữ liệu mã trong ngành này." · Back button hoạt động
marketCap = null (tab Vốn hóa) LOW API 2 Exclude sector khỏi treemap hoặc hiển thị "—" · Không crash
Market closed / weekend LOW — Banner "Dữ liệu phiên gần nhất: dd/MM/yyyy" · Không poll
sizeMetric = 0 / null cho nhiều sectors LOW — Set min value = 0.3% tổng sum · Ô vẫn xuất hiện, ẩn label · Không exclude
5 Quick Reference — API Endpoints
GET /api/v1/marketWatch/sectorIndex API 1

Request params:

tradingDate — yyyy-MM-dd HH:mm:ss (optional, default = hôm nay)

sortOrder — VSTOCK_ASC | VSTOCK_DESC | CHANGE_ASC | CHANGE_DESC

Key response fields:

sectorId · sectorName · perChange · tradingValue · tradingVolume · foreignBuyVolume · foreignSellVolume

GET /api/v1/marketWatch/sectorDetail API 2

Request params:

Không có param từ FE. BE tự hardcode languageID=1.

Key response fields:

sectorId · sectorLevel · marketCap · pe · pb · perChange1W · perChange1M · perChange3M

GET /api/v1/marketWatch/sectorStocks?sectorId={id} API 3

Request params:

sectorId — Required · lấy từ sectorIndex.sectorId

Key response fields:

stockCode · stockName · exchange · perChange · tradingValue · tradingVolume · marketCap · foreignBuyVolume · foreignSellVolume

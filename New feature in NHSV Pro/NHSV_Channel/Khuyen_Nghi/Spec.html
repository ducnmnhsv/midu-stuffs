  • 3 pills: Tất cả (default) / Bluechip / Midcap. Search box theo mã CK ở góc phải.

5. API Specification — Mobile App

Base URL: /api/v1 · Auth: Bearer token
GET /recommendations/portfolio Danh mục cơ bản — list với giá realtime enriched
Query Parameters
segmentstring?BLUECHIP | MIDCAP. Null → tất cả
stockCodestring?Search theo mã CK (LIKE)
fetchCountint?Số bản ghi mỗi trang. Default: 20
lastRecommendationIdstring?Cursor phân trang. Trang đầu bỏ qua; trang sau = recommendationId cuối cùng của trang trước
Response (200) — Recommendation item
recommendationIdstringID khuyến nghị
stockCodestringMã CK
segmentstring"BLUECHIP" | "MIDCAP"
targetPricenumberGiá kỳ vọng (VNĐ)
currentPricenumber?Giá hiện tại — từ market data service. Null nếu không fetch được.
expectedReturnPercentnumber?% tiềm năng = (targetPrice/currentPrice-1)×100. Null nếu currentPrice null.
recommendationDatedateISO date
hasPdfboolCó báo cáo PDF không
Price enrichment: BE join với market data service khi serve API. Nếu market data không available (ngoài giờ giao dịch): dùng closing price phiên gần nhất.
Response wrapper: {"totalCount": ..., "recommendations": [...]}. Pagination theo cursor fetchCount + lastRecommendationId — nhất quán với convention TradeX-native khác (vd. stopOrder history), thay cho page/pageSize.
Filter status: Chỉ trả records có status = 'ACTIVE'.

Đã bỏ page/pageSize khỏi bản draft trước — vừa lệch convention pagination, vừa mâu thuẫn với "filter client-side" của Screen 3.3.

1
Admin nhận file Excel từ Phòng QTRR
File Excel có ít nhất 4 cột bắt buộc: Mã, Cơ bản, Thị trường, NHSV Rating. Giá trị hợp lệ: S, A, B, C, D (case-insensitive).
2
Admin upload lên nhsv-admin
Gọi POST /admin/recommendations/ratingUpload với file Excel (multipart). BE parse → validate → return preview (số records hợp lệ, lỗi nếu có).
3
Admin xem preview và confirm
Admin FE hiển thị preview table (10 rows đầu + tổng số records). Nếu OK → Admin bấm "Xác nhận publish". Nếu có lỗi → hiển thị danh sách mã lỗi → Admin fix file và upload lại.
4
BE tạo batch mới và activate
Gọi POST /admin/recommendations/ratingPublish với batchId từ bước 2. BE: INSERT tất cả records vào nhsv_ratings với batch_id mới → SET batch.is_active = true → SET cũ is_active = false. Nguyên tử (transaction).
5
App đọc batch mới ngay lập tức
Không cần cache clear vì API luôn query is_active = true trong real-time.

📅 Kỹ thuật hằng ngày — Draft → Publish Flow (mới — 2026-07-10)
Admin chọn 1 ngày giao dịch, nhập nhiều mã trong cùng 1 form (mỗi dòng = 1 mã), lưu 1 lần → tất cả vào DB với status='DRAFT'. App mobile không thấy record DRAFT. Khi Phòng PT xong việc soạn cho ngày đó, bấm "Publish ngày [tradeDate]" → toàn bộ DRAFT của ngày đó chuyển PUBLISHED cùng lúc, atomic. Có thể publish nhiều lần/ngày nếu thêm mã mới sau đó (chỉ publish phần DRAFT còn lại, không ảnh hưởng record đã PUBLISHED trước đó).

7. Admin Screen Specs tại tnhsvpro.nhsv.vn/nhsv-admin

📋 Admin Screen 1 — Danh mục cơ bản Management
URL: /nhsv-admin/recommendations/portfolio
  • Table: Mã CK | Segment | Giá KN | Giá Kỳ vọng | Ngày KN | PDF | Trạng thái | Thao tác
  • Filter: Bluechip/Midcap dropdown + ô search theo mã + status filter (Active/Archived)
  • Nút "+ Thêm khuyến nghị" → form tạo mới
  • Thao tác: Sửa | Archive (không xóa cứng)
✏️ Admin Screen 2 — Form Tạo/Sửa Danh mục
URL: /nhsv-admin/recommendations/portfolio/new
  • Mã CK: input với autocomplete từ danh mục (required)
  • Segment: Bluechip / Midcap dropdown (required)
  • Giá tại ngày KN: số (required)
  • Giá kỳ vọng: số, auto tính % tiềm năng hiển thị live khi nhập (required)
  • Ngày khuyến nghị: date picker (required)
  • Ngày hết hiệu lực: date picker (optional) — để trống = không giới hạn thời gian, khuyến nghị không tự Archive (Q1 resolved 2026-07-10)
  • Luận điểm: textarea (optional)
  • Upload PDF: drag-drop, .pdf only (optional)
📊 Admin Screen 3 — NHSV Rating Upload
URL: /nhsv-admin/recommendations/rating
  • Header: "NHSV Rating" + badge "Batch hiện tại: MM/YYYY · N mã"
  • Upload area: drag-drop file .xlsx / .xls. Required columns tooltip: "Mã, Cơ bản, Thị trường, NHSV Rating"
  • Sau upload: preview table 10 rows + tổng records + error list (nếu có)
  • Nút "Xác nhận publish": active nếu không có errors. Sau confirm → toast "Đã cập nhật N mã rating"
  • Lịch sử: bảng các batch cũ (ngày upload, số mã, người upload)
📅 Admin Screen 4 — Kỹ thuật hằng ngày
URL: /nhsv-admin/recommendations/daily-technical
  • Date picker để chọn ngày (default: hôm nay) — chọn được bất kỳ ngày trong quá khứ, không giới hạn hôm nay. Hiển thị toàn bộ entry của ngày đó, gồm cả Nháp và đã Publish.
  • Ngày không có entry nào (kể cả Nháp): empty state rõ ràng "Ngày [X] không có mã kỹ thuật nào — kể cả Nháp" — không hiểu nhầm là lỗi tải dữ liệu.
  • Table: Mã CK | Vùng mua | Mục tiêu | Cắt lỗ | Upsize | PDF | Nguồn | Trạng thái (Nháp/Còn hiệu lực/Đạt target) | Thao tác
  • Badge "MANUAL" / "AUTO" trên cột Nguồn — phân biệt entry thủ công vs tương lai tự động
  • Nút "+ Thêm entry" (đổi hành vi — 2026-07-10): mở form batch — chọn 1 Ngày áp dụng, sau đó thêm nhiều dòng mã (mỗi dòng: Mã CK, Vùng mua from/to, Mục tiêu, Cắt lỗ, Upsize optional) qua nút "+ Thêm dòng". Bấm "Lưu nháp" → tất cả lưu status=DRAFT, chưa hiện trên app.
  • Nút "Publish ngày [ngày đang chọn]" (mới): hiện/enable khi ngày đang chọn có ≥ 1 entry Nháp. Bấm → toàn bộ Nháp của ngày đó chuyển "Còn hiệu lực" cùng lúc, app đọc thấy ngay.
  • Sửa từng dòng (action Edit riêng): dùng khi cần bổ sung Luận điểm/PDF sau khi batch-tạo, hoặc chỉnh số liệu — áp dụng cho cả entry Nháp và đã Publish.
  • Delete: hard delete (không soft delete — dữ liệu kỹ thuật ngắn hạn), áp dụng cho cả Nháp và đã Publish.

9. Open Questions

Cần xác nhận trước khi estimate / bắt đầu dev
✓
[RESOLVED 2026-07-10] Danh mục cơ bản: khuyến nghị có expire hay tự quản lý?
Trả lời: Có expire, set theo từng mã (optional, field expiry_date). Không set = không tự hết hạn, admin tự Archive tay. Có set = scheduled job (BE-11) tự SET status='ARCHIVED' sau ngày hết hiệu lực. Đã cập nhật data model (mục 4), Admin form (mục 7), Work Breakdown (mục 8).
2
[Non-blocking] Kỹ thuật hằng ngày: có cần xem lại các ngày trước không?
Spec hiện tại: default load hôm nay, app có thể pass tradeDate để xem ngày khác. Nhưng UI hiện chỉ có "Hôm nay" badge, không có date picker. Cần quyết định: (a) Chỉ hiện hôm nay (đơn giản nhất); (b) Cho phép xem lại lịch sử qua date picker. Nếu (b) → cần thêm date picker UI trên mobile.
❓ Owner: PM
3
[Non-blocking] Automation pipeline Vietstock → Kỹ thuật hằng ngày: timeline?
Spec đã để sẵn dataSource: 'VIETSTOCK_AUTO' field và endpoint đã accept automation. Nhưng cần xác định: Vietstock gửi data theo channel nào (email, SFTP, API)? Ai xử lý file raw → format đúng schema? Khi nào bắt đầu build pipeline? Hiện tại là v1 manual — automation là v2.

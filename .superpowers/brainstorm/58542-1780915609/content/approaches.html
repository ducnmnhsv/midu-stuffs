GTGD Chart — 3 Approaches

Vấn đề: MongoDB chỉ có giá trị cuối ngày, cần per-minute data để vẽ "GTGD phiên trước"

A

Extend EOD Flush — lưu per-minute GTGD vào MongoDB

BE: Sửa job flush 15:30 trong realtime-v2: thay vì bỏ, ghi toàn bộ time-series GTGD theo phút vào collection mới symbolValueMinutes hoặc thêm field cv (cumulative value) vào symbolQuoteMinutes.

API: Extend /tradingview/history — khi from/to là ngày hôm qua, query MongoDB symbolValueMinutes trả về đúng format { t, v }.

FE: Gọi API 2 lần — một cho hôm nay, một cho phiên trước (from/to = ngày hôm qua). WebSocket cập nhật today line real-time.

Pros

  • Tận dụng API đã có, FE thay đổi ít
  • Consistent với TradingView format
  • Data lưu vĩnh viễn, có thể query nhiều ngày

Cons

  • Cần sửa realtime-v2 (Java)
  • Chưa có data cho các ngày trước khi deploy
  • Job flush cần chạy đúng, nếu miss = mất data

⭐ Recommended — Bền vững nhất, tái sử dụng infrastructure

B

Redis Key Preservation — giữ key hôm qua trong 24h

BE: Trước khi clear Redis key SYMBOL_QUOTE_MINUTE_{code} lúc EOD, copy nó sang key mới SYMBOL_QUOTE_MINUTE_{code}_PREV với TTL 24h.

API: Thêm param prevSession=true vào /tradingview/history → query từ Redis key _PREV.

FE: Gọi 2 API: 1 normal + 1 với prevSession=true.

Pros

  • Implement nhanh nhất
  • Không cần MongoDB schema change
  • Data có ngay từ ngày đầu deploy

Cons

  • Chỉ có T-1, không có lịch sử dài hơn
  • Redis TTL: nếu server restart mất data
  • Tăng Redis memory (double key)
C

New Dedicated API — tách riêng endpoint GTGD

BE: Tạo endpoint mới GET /market/gtgd/history nhận symbol + date. Logic: nếu date = today → query Redis; nếu date = trước → query MongoDB symbolValueMinutes (cần tạo collection này).

FE: Component GtgdAreaChart gọi API mới, không phụ thuộc tradingview/history.

Pros

  • API clean, đúng purpose
  • Response format tối ưu cho chart (không cần OHLC)
  • Không ảnh hưởng TradingView flow

Cons

  • Nhiều effort nhất — thêm endpoint mới hoàn toàn
  • Vẫn cần sửa realtime-v2 để lưu per-minute GTGD
  • FE cần integrate API mới thay vì tái dùng existing

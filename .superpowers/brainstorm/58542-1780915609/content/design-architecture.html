Design — Section 1: Architecture & Data Flow

Approach A: Extend EOD Flush để lưu per-minute cumulative GTGD vào MongoDB

Vấn đề hiện tại

✗ TODAY Redis SYMBOL_QUOTE_MINUTE_{code} → /tradingview/history → FE (✅ works)
✗ PREV MongoDB symbolQuoteMinutes → chỉ có daily total → không có per-minute

Flow sau khi implement

📊 GTGD hôm nay (real-time)
WebSocket market.quote.{symbol} → field va (cumulative GTGD)
  ↓
FE subscribes → push to today series (real-time update)
  +
GET /tradingview/history?from=todayOpen&to=now
  → realtime-v2 Redis SYMBOL_QUOTE_MINUTE_{code}
  → returns { t[], v[] } (v = cumulative GTGD per minute)
📈 GTGD phiên trước (static)
GET /tradingview/history?from=prevOpen&to=prevClose
  → market-query-v2 MongoDB symbolQuoteMinutes
  → NEW: query field cv (cumulative value) → map to response v
⚙️ EOD Flush Job (15:30 daily) — KEY CHANGE
realtime-v2 FlushJob:
  1. Read Redis SYMBOL_QUOTE_MINUTE_{code} (all today's bars)
  2. NEW: include tradingValue (= cumulative GTGD) khi save
  3. Write to MongoDB symbolQuoteMinutes với field cv
  4. Clear Redis key (như hiện tại)

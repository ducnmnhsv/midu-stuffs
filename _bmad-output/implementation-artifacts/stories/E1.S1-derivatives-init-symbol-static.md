# E1.S1: Daily init adds derivatives symbols to symbol_static.json

## User Story
As a system (daily scheduler)
I want to refresh the list of derivatives symbols at the start of each trading day
So that the app can display derivatives contracts with correct static/reference information before market open.

## Acceptance Criteria
- [ ] AC-01: On trading days, the daily init process includes VN30 futures symbols in the generated `symbol_static.json`.
- [ ] AC-02: Derivatives entries in `symbol_static.json` follow the existing format and include `m = "derivatives"` to distinguish them from equity.
- [ ] AC-03: If derivatives symbol retrieval fails, the init job still completes for equity (no full job failure).
- [ ] AC-04: For each derivatives symbol, the output includes required fields: `s, m, n1, n2, t, re, ce, fl, lq, bc, ed, rd`.

## Scope Notes (Business)
- The list of derivatives symbols is sourced from Lotte market data.
- Derivatives must be additive to existing equity behavior.

## Output Contract (symbol_static.json)
Derivatives must follow the same template as equity:

```json
{
    "s": "VN30F2501",
    "m": "derivatives",
    "n1": "HĐ Tương lai VN30 Tháng 01/2025",
    "n2": "VN30 Index Futures Jan 2025",
    "t": "FUTURES",
    "re": 1273.0,
    "ce": 1350.0,
    "fl": 1220.0,
    "lq": 0,
    "bc": "VN30",
    "ed": "20250130",
    "rd": 15
}
```

## Technical Notes (minimal)
- Integration points: Lotte APIs `/tuxsvc/market/dr/stock-board`, `/tuxsvc/market/dr/stock-price`
- Primary artifact: `symbol_static.json`

## Test Scenarios
| TC ID | Scenario | Expected |
|-------|----------|----------|
| TC-01 | Trading day init completes | `symbol_static.json` contains derivatives entries with `m="derivatives"` |
| TC-02 | Derivatives retrieval fails | Equity init completes; derivatives missing; warning logged |
| TC-03 | Partial derivatives detail missing | Only valid derivatives are included; job still completes |

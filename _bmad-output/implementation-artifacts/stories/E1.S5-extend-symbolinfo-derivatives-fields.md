# E1.S5: Extend SymbolInfo to support derivatives-specific fields (backward compatible)

## User Story
As a backend platform
I want SymbolInfo to include derivatives-specific fields
So that futures contracts can be represented consistently across init, aggregation, API, and WebSocket.

## Acceptance Criteria
- [ ] AC-01: SymbolInfo supports storing derivatives fields: open interest, base code, expiry date, remaining days, theoretical price, basis (as applicable).
- [ ] AC-02: Existing equity symbols remain backward compatible (derivatives fields are optional / empty for equity).
- [ ] AC-03: Derivatives identification uses `m="derivatives"` and `t="FUTURES"` consistently across outputs.

## Business Rules
- **BR-01 (Backward compatibility)**: Adding derivatives fields must not break existing equity consumers.
- **BR-02 (Consistency)**: Field naming and meaning are consistent across `symbol_static.json`, `/api/v2/market/symbolInfo`, and WebSocket payloads.

## Data Fields (Business)
| Field | Meaning | Example |
|------|---------|---------|
| `oi` | Open Interest (contracts open) | 45000 |
| `bc` | Base code (underlying) | VN30 |
| `ed` | Expiry date (yyyyMMdd) | 20250130 |
| `rd` | Remaining days | 15 |
| `tp` | Theoretical price (if available) | 1280.0 |
| `bs` | Basis (if available) | -5.0 |

## Technical Notes (minimal)
- Applies to SymbolInfo representation used by init + aggregation + API + WS
- Derivatives fields are additive (optional)

## Test Scenarios
| TC ID | Scenario | Expected |

| TC-02 | Derivatives symbol storage | Derivatives fields present and retrievable |

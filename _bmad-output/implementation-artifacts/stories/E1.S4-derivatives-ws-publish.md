# E1.S4: Publish derivatives updates via TradeX WebSocket channels

## User Story
As a mobile/web client
I want to subscribe to TradeX WebSocket channels for derivatives symbols
So that I receive real-time futures price and order book updates on the app.

## Acceptance Criteria
- [ ] AC-01: Clients can subscribe to derivatives quote channels using pattern `market.quote.dr.{code}`.
- [ ] AC-02: Clients can subscribe to derivatives bid/offer channels using pattern `market.bidoffer.dr.{code}`.
- [ ] AC-03: Messages include `m="derivatives"` and follow the same field conventions as equity wherever applicable.
- [ ] AC-04: Equity WebSocket channels remain unchanged and unaffected.

## Business Rules
- **BR-01 (Minimize FE changes)**: Message shape should align with existing equity consumer expectations; only derivatives-specific fields are additive.
- **BR-02 (Isolation)**: Failures in derivatives publishing must not impact equity channels.

## Message Example (Quote)
```json
{
  "channel": "market.quote.dr.VN30F2501",
  "data": {
    "s": "VN30F2501",
    "m": "derivatives",
    "c": 1286.0,
    "ch": 13.0,
    "ra": 1.02,
    "vo": 125500,
    "tb": 5100,
    "to": 4600,
    "fr": {"bv": 5200, "sv": 3100},
    "oi": 45100
  }
}
```

## Flow
```
Derivatives updates available (from aggregation pipeline)
  └─→ Determine channel name by symbol
        ├─→ Emit quote updates to market.quote.dr.{code}
        └─→ Emit bidoffer updates to market.bidoffer.dr.{code}
Equity channels continue unchanged
```

## Technical Notes (minimal)
- Client-facing channels:
  - Quote: `market.quote.dr.{code}`
  - Bid/Offer: `market.bidoffer.dr.{code}`
- Upstream dependency: aggregation / update stream from E1.S3

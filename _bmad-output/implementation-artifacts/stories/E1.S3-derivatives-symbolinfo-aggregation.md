# E1.S3: Aggregate derivatives data into SymbolInfo cache for /api/v2/market/symbolInfo

## User Story
As a backend market data platform
I want derivatives real-time updates to be aggregated into the same SymbolInfo cache used by equity
So that `/api/v2/market/symbolInfo` can return derivatives data without requiring FE flow changes.

## Acceptance Criteria
- [ ] AC-01: Derivatives symbols exist in SymbolInfo cache after daily init (static/reference fields available).
- [ ] AC-02: During trading sessions, derivatives real-time updates enrich the cached SymbolInfo record (latest price + bid/offer).
- [ ] AC-03: `/api/v2/market/symbolInfo` returns derivatives symbols with `m="derivatives"` and the latest aggregated fields.
- [ ] AC-04: If a derivatives update arrives for a symbol not present in cache, it is skipped with a warning (no crash).
- [ ] AC-05: Equity SymbolInfo aggregation remains unchanged and stable.

## Business Rules
- **BR-01 (Minimize FE changes)**: FE should keep calling `/api/v2/market/symbolInfo` as today; derivatives must “just appear” via aggregation.
- **BR-02 (Data completeness)**: Cache must preserve static fields from init (e.g., reference prices, expiry) while updating dynamic fields from WS.
- **BR-03 (Consistency)**: Updates should not corrupt or erase unrelated fields (e.g., expiry fields).

## Technical Notes (minimal)
- Data source: derivatives ingestion pipeline (E1.S2)
- Cache location: SymbolInfo cache used by existing equity API (`/api/v2/market/symbolInfo`)
- Dependency: derivatives symbols must be initialized (E1.S1)

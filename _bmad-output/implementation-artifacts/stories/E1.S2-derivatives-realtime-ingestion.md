# E1.S2: Ingest derivatives real-time data from Lotte WebSocket

## User Story
As a market data system
I want to receive derivatives quote and bid/offer updates in real time
So that derivatives prices and order book can be updated continuously during trading sessions.

## Acceptance Criteria
- [ ] AC-01: The system subscribes to derivatives real-time channels for all active futures symbols.
- [ ] AC-02: Quote updates and bid/offer updates are ingested continuously during trading sessions.
- [ ] AC-03: If derivatives WS disconnects or receives malformed messages, equity real-time processing is not impacted.
- [ ] AC-04: The ingestion pipeline emits structured updates downstream for aggregation (no FE dependency).

## Business Rules
- **BR-01 (Isolation)**: Derivatives ingestion failures must not degrade equity ingestion.
- **BR-02 (Latency)**: Derivatives updates should be processed fast enough to support near real-time UX (target < 1s end-to-end).
- **BR-03 (Resilience)**: Malformed messages are skipped with warning logs; ingestion continues.

## Flow
```
Connect to Lotte WS
  └─→ Subscribe derivatives channels for all VN30F codes
        ├─→ Receive quote updates → transform to internal update → forward to aggregation
        └─→ Receive bid/offer updates → transform to internal update → forward to aggregation
If error: log warning/error → continue (and reconnect if applicable)
```

## Technical Notes (minimal)
- Source: Lotte derivatives WebSocket channels
  - Quote: `pro.pub.auto.dr.qt./{code}`
  - Bid/Offer: `pro.pub.auto.dr.bo./{code}`
- Downstream: events forwarded for aggregation (e.g., via messaging topics dedicated to derivatives)

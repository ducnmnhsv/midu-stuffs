eKYC (Electronic Know Your Customer) Database Schema

This document describes the database schema for the eKYC domain.

Overview

The eKYC domain handles electronic KYC processing, customer verification, contract management, and related supporting data. It processes identity documents, verifies customer information, and manages electronic contracts.

Entity Relationship Diagram

Core Entities

e_kyc

The main eKYC record containing customer identity information, document images, and verification status.

Key Fields: - identifier_id: National ID or passport number - full_name: Customer’s full name - front_image_url, back_image_url: Identity document images - portrait_image_url: Customer portrait photo - matching_rate: OCR and face matching confidence score - status: KYC processing status - contract_id: Associated contract reference

Key Relationships: - One-to-one with ekyc_ext (extended information) - One-to-one with ekyc_additional_info (additional details) - One-to-many with econtract (generates contracts) - One-to-many with ekyc_creator_status (tracks creator status)

ekyc_ext

Extended eKYC information for additional fields not in the main table.

Key Relationships: - Many-to-one with e_kyc (extends main record)

ekyc_additional_info

Additional information related to the eKYC record (e.g., employment, income).

Key Relationships: - Many-to-one with e_kyc (belongs to eKYC record)

ekyc_creator_status

Tracks the status of eKYC creation process and creator information.

Key Relationships: - Many-to-one with e_kyc (tracks eKYC record)

econtract

Electronic contracts generated from eKYC records.

Key Relationships: - Many-to-one with e_kyc (generated from eKYC) - One-to-many with econtract_info (contains contract details) - One-to-many with econtract_info_history (tracks changes)

econtract_info

Detailed information within electronic contracts.

Key Relationships: - Many-to-one with econtract (belongs to contract) - One-to-many with econtract_info_history (tracks changes)

econtract_info_history

Historical changes to contract information for audit purposes.

Key Relationships: - Many-to-one with econtract_info (tracks contract info changes)

Supporting Entities

ekyc_bank_list

List of banks and branches available for account setup.

Usage: Referenced during eKYC process for bank account information.

ttl_issue_place_code_map

Mapping of issue place codes to locations (for identity documents).

Usage: Used to validate and map issue places from identity documents.

matching_rate

OCR and face matching rate records for verification.

Usage: Stores matching confidence scores for identity verification.

blockholder

List of blocked/blacklisted individuals.

Usage: Checked during eKYC to prevent blocked individuals from registering.

public_coop

Public cooperation/partnership information.

Usage: Referenced during eKYC for partnership-related processing.

eKYC Processing Flow

Document Upload → OCR Processing → Face Matching → Verification → Contract Generation
  1. Document Upload: Customer uploads identity documents (front, back, portrait)
  2. OCR Processing: System extracts data from documents
  3. Face Matching: Portrait is matched against document photo
  4. Verification:
    • Check against blockholder list
    • Validate against ttl_issue_place_code_map
    • Calculate matching_rate
  5. Contract Generation: If verified, generate econtract and econtract_info

Contract Management Flow

eKYC → Contract Creation → Contract Info → Contract History
  1. eKYC record is approved
  2. Electronic contract is created (econtract)
  3. Contract details are stored (econtract_info)
  4. All changes are tracked (econtract_info_history)

Status Tracking

  • e_kyc.status: Overall KYC status (pending, approved, rejected, etc.)
  • ekyc_creator_status.status: Creator-specific status
  • econtract.status: Contract status (draft, signed, active, expired)
  • econtract_info_history: Tracks all contract information changes

Data Validation

  • Identifier Validation: Checks identifier_id against blockholder
  • Issue Place Validation: Validates against ttl_issue_place_code_map
  • Bank Validation: Validates bank information against ekyc_bank_list
  • Matching Rate: Uses matching_rate for verification confidence

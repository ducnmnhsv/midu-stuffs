VietStock Bridge Database Schema

This document describes the database schema for the VietStock Bridge domain.

Overview

The VietStock Bridge domain provides integration with VietStock data sources for stock market information, news, and financial data. It manages stock news, stock-symbol relationships, and provides a bridge between the NHSV platform and external VietStock APIs.

Entity Relationship Diagram

Field Descriptions

t_stock_news

  • id: Primary key identifier for the news article (part of composite key)
  • language: Language code (e.g., “vi”, “en”) - part of composite key
  • title: Title of the stock news article
  • head: Headline or summary of the news article
  • url: Original URL of the news article from VietStock
  • publish_time: Publication timestamp of the news article
  • content: Full content of the news article
  • author: Author of the news article
  • source: Source of the news article

t_stock_stock_news

  • stock_news_id: Foreign key to the stock news article (part of composite key)
  • stock_code: Stock symbol code (e.g., “VIC”, “VNM”) - part of composite key

Core Entities

t_stock_news

Stock news articles from VietStock. Stores news content, metadata, and publication information.

Key Fields: - id: News article ID (part of composite primary key) - language: Language code (part of composite primary key) - title: News article title - head: News headline/summary - url: Original article URL - publish_time: Publication timestamp - content: Full article content - author: Article author - source: News source

Key Relationships: - One-to-many with t_stock_stock_news (news can be linked to multiple stocks)

t_stock_stock_news

Junction table linking stock news articles to stock symbols. This allows multiple stocks to be associated with a single news article.

Key Fields: - stock_news_id: Foreign key to t_stock_news.id - stock_code: Stock symbol code (e.g., “VIC”, “VNM”)

Key Relationships: - Many-to-one with t_stock_news (belongs to news article)

Data Flow

News Aggregation Flow

VietStock API → Stock News → Stock-Symbol Mapping
  1. News Collection: VietStock Bridge fetches news from VietStock API
  2. News Storage: News articles are stored in t_stock_news with language support
  3. Symbol Linking: News articles are linked to relevant stock symbols via t_stock_stock_news
  4. Data Access: Other services query news by stock symbol or news ID

Multi-Language Support

The t_stock_news table uses a composite primary key (id, language) to support multiple language versions of the same news article. This allows: - Same news article in different languages (Vietnamese, English, etc.) - Language-specific content and translations - Unified news ID across languages

Stock Symbol Relationships

The t_stock_stock_news table enables: - One-to-Many: One news article can be associated with multiple stocks - Many-to-One: Multiple news articles can reference the same stock - Flexible Tagging: News can be tagged with relevant stock symbols for filtering

Integration Points

VietStock API Integration

  • Fetches real-time stock news
  • Retrieves company profiles and financial data
  • Provides stock event information
  • Supplies transaction history data

Data Storage

  • MySQL for relational data (news, stock relationships)
  • MongoDB for document-based data (financial statements, symbol info, watchlists)

Data Access

  • RESTful API endpoints for news queries
  • Stock symbol-based filtering
  • Language-specific content retrieval
  • Time-based filtering (publish_time)

Supporting Data Models

While the database schema shows only two tables, the VietStock Bridge also uses MongoDB collections for: - ISymbolInfo: Stock symbol information and real-time quotes - IFinancialStatement: Financial statements (quarterly and yearly) - IWatchList: User watchlists

These are stored in MongoDB for better performance with document-based queries and are not included in the MySQL schema.

Usage Patterns

Querying News by Stock

SELECT sn.* 
FROM t_stock_news sn
JOIN t_stock_stock_news ssn ON sn.id = ssn.stock_news_id
WHERE ssn.stock_code = 'VIC' AND sn.language = 'vi'
ORDER BY sn.publish_time DESC;

Querying All Stocks for News

SELECT ssn.stock_code
FROM t_stock_stock_news ssn
WHERE ssn.stock_news_id = 12345;

Notes

  • The schema uses composite primary keys for multi-language support
  • News articles are linked to stocks through a junction table for flexibility
  • The bridge service acts as a data aggregator and translator between VietStock APIs and NHSV services
  • Real-time stock data is primarily stored in MongoDB for performance
  • The MySQL schema focuses on relational data (news and relationships)

Flow Overview

Order 2FA chạy theo ngưỡng giá trị giao dịch. Giao dịch nhỏ (<10tr) không cần 2FA — không làm phiền user. Giao dịch lớn tự động chọn phương thức mạnh nhất có sẵn.

Bước 1
Check giá trị
User chọn orders → tap "Confirm" → hệ thống tự động check tổng giá trị — hoàn toàn tự động, không popup
FE
Bước 2
Chọn phương thức
<10tr: không cần 2FA (silent)
10-100tr: Biometric L2 (Face ID)
≥100tr: Smart-OTP TOTP
BE + FE
Bước 3
Xác thực
User xác thực bằng biometric hoặc nhập OTP — tối đa 5 lần thử, fallback SMS nếu thiết bị chính không khả dụng
FE
Bước 4
Gửi lệnh + 2FA token
BE verify proof token, kiểm tra transactionHash match — 1-time token, 60s TTL, không replay được

UX Principles

Order 2FA cần bảo mật nhưng không làm chậm trải nghiệm giao dịch. Mục tiêu: chỉ can thiệp khi cần, tối ưu cho giao dịch thường xuyên.

🔄

Tự động chọn method

Không bắt user chọn phương thức 2FA — hệ thống tự động chọn method mạnh nhất có sẵn dựa trên giá trị giao dịch. Giao dịch nhỏ không cần 2FA

⬇️

Fallback mượt mà

Nếu biometric/Smart-OTP không khả dụng, tự động fallback SMS OTP — không block giao dịch, không yêu cầu user cài đặt thêm

🔒

Minh bạch về bảo mật

Hiển thị rõ lý do cần 2FA ("Giao dịch ≥ 100tr cần xác thực OTP"), số tiền, SĐT nhận OTP — user hiểu tại sao và yên tâm

Background & Current Gaps

TT134 Điều 10 yêu cầu xác thực giao dịch 2 lớp. Hiện tại chỉ có access token — chưa đủ.

Current
  • Confirm chỉ dùng access token — không có 2FA
  • Smart-OTP đang implement (chưa xong)
  • Biometric L2 chưa có (chỉ có L1 login)
  • Không phân biệt giao dịch nhỏ/lớn
  • Không có proof token — không chống replay
Target
  • 2FA theo ngưỡng: <10tr không 2FA, 10-100tr Biometric L2, ≥100tr Smart-OTP
  • 3 phương thức: Smart-OTP TOTP, Biometric L2, SMS OTP fallback
  • 2FA proof token: JWT 60s, single-use, transactionHash-bound
  • Lockout sau 5 lần fail — chống brute force
  • Tích hợp vào ConfirmOrdersScreen hiện tại

03👥

BE / FE Responsibilities

Phân công rõ ràng — BE xử lý challenge/verify/proof token, FE xây dựng UX confirm + OTP/biometric prompt.

Backend (AAA Service)
  • 2FA Challenge System — t_2fa_challenge table, create/verify
  • POST /transactions/2fa/challenge — khởi tạo 2FA
  • POST /transactions/2fa/verify — verify + trả proof token
  • Proof token generation — JWT 60s, single-use, transactionHash-bound
  • require2FA middleware — check X-2FA-Token trên endpoints confirm
  • Smart-OTP proxy — verify TOTP qua Lotte Core
  • Biometric L2 verify — RSA signature verify
  • SMS OTP send/verify — rate limit, cooldown, mask phone
  • Lockout logic — 5 fails → lock 30 phút
Frontend (App)
  • TwoFAChallengeModal — modal chọn phương thức 2FA
  • SmartOTPInput — 6-digit OTP input component
  • SMSOTPInput — OTP input + resend button + countdown
  • BiometricPromptHandler — Face ID / Touch ID prompt wrapper
  • TwoFAProviderSelector — chọn method nếu có nhiều option
  • ConfirmOrdersScreen update — thêm 2FA flow vào saga
  • Cash/Stock/Loan sagas — thêm 2FA check tương tự
  • Error handling — retry, fallback SMS, lockout notification
04⚙️

2FA Tier Rules

3-tier system — tự động chọn mức xác thực dựa trên giá trị giao dịch và phương thức có sẵn.

Giá trị giao dịch 2FA Required Phương thức ưu tiên Fallback
< 10 triệu VND ❌ Không — (access token đủ) —
10 triệu → 100 triệu ✅ Có Biometric L2 (Face ID / Touch ID) SMS OTP nếu biometric unavailable
≥ 100 triệu VND ✅ Có Smart-OTP TOTP SMS OTP nếu Smart-OTP chưa active
📋 Transaction types cần 2FA: Order (đặt lệnh theo ngưỡng), Cash transfer (mọi giá trị), Stock transfer (mọi giá trị), Profile change (SĐT/email/password), Biometric enrollment.

08🔑

2FA Proof Token

JWT ngắn hạn, 1-time, bound to transaction — chống replay và transaction mismatch.

Token payload BE

{
  "sub": "2fa_proof",
  "userId": 12345,
  "transactionType": "ORDER_CONFIRM",
  "transactionHash": "sha256(...)",
  "method": "SMARTOTP",
  "iat": 1718200000,
  "exp": 1718200060   // 60s TTL
}

Verification flow BE

  • require2FA middleware — check header X-2FA-Token
  • Verify JWT — signature, expiry, sub=2fa_proof
  • Transaction match — SHA256(request body + userId + challengeId)
  • Single-use — revoke jti ngay sau khi verify
  • 60s TTL — từ lúc verify thành công đến lúc dùng

Open Questions

Các quyết định cần PM/compliance confirm.

1
Threshold 10tr / 100tr có phù hợp?
10tr cho Biometric L2, 100tr cho Smart-OTP. Cần compliance confirm các ngưỡng này đúng với TT134.
Cần: Compliance
2
User chưa enroll Smart-OTP: force enroll hay SMS fallback?
Đề xuất: SMS fallback (Phase 1), sau đó force enroll Smart-OTP (Phase 2 — khi user thấy tiện lợi hơn SMS)
Cần: PM + Product
3
Admin transactions có cần 2FA không?
Có — nhưng admin có thể bypass (configurable). Một số giao dịch admin nội bộ có thể skip 2FA.
Cần: Compliance
4
Biometric L2 verify local (AAA) hay qua Lotte Core?
Verify local trên AAA (RSA signature với public key từ enrollment) — không cần gọi Lotte Core cho biometric.
Quyết định: Local verify

STT14b — Rút tiền ≥10M (Biometric Per-transaction) P2
Điều 12k4a · BLOCKED chờ GATE (C06 vs VNPT) · stt14b
Giữ nguyên model cũ. Biometric server-side per-transaction, verify qua kbfinance/verifyPassword (pinType=BIOMETRIC). Không cho fallback Smart OTP. Nếu chưa enroll biometric → block giao dịch. Blocked chờ GATE decision (C06 vs VNPT).
BE Tasks
BE-1
POST /api/v1/transactions/2fa/challenge: tạo challenge cho rút tiền ≥10M với transactionType: WITHDRAWAL, trả challengeId + transactionHash
challengeId TTL: 5 phút. 1 challenge active per user tại 1 thời điểm.
CRITICAL
BE-2
POST /api/v1/transactions/2fa/verify: nhận RSA signature từ FE → verify → gọi kbfinance/verifyPassword (pinType=BIOMETRIC) → trả proof token (JWT 60s, 1-time)
Dùng verifyPwdBiometric() đã có trong BiometricService.ts
CRITICAL
BE-3
Middleware X-2FA-Token cho /cash/withdraw/confirm: verify proof token → match transactionHash → revoke → process. Log auth_method=BIOMETRIC, auth_time
BE-4
Guard: withdrawal ≥10M và không có X-2FA-Token → reject BIOMETRIC_AUTH_REQUIRED
FE Tasks
FE-1
Detect withdrawal amount ≥10M → gọi challenge API, trigger BiometricPromptHandler (Face ID/Touch ID)
FE-2
Ký challengeId + transactionHash bằng RSA private key từ biometric enrollment → gửi lên verify API → nhận proof token → gọi withdraw/confirm kèm X-2FA-Token
FE-3
Chưa enroll biometric + withdrawal ≥10M → show error rõ ràng yêu cầu enroll biometric. Không show OTP fallback.
CRITICAL

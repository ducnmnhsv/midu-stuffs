Background & Current Gaps

TT134 yêu cầu định danh thiết bị để hỗ trợ phát hiện bất thường và xác thực giao dịch.

Current
  • Dùng deviceId từ client — dễ giả mạo
  • Không có chuẩn — mỗi service lưu device riêng
  • WTS (macOS) không có device ID ổn định
  • Không phát hiện device bất thường
Target
  • Fingerprint hash server-side — không thể spoof
  • 1 chuẩn chung, mọi service dùng chung
  • WTS: dùng IP /24 + canvas fingerprint
  • Cảnh báo khi device lạ login

System Architecture

Dữ liệu từ client → rest-proxy → AAA Service → MySQL. Fingerprint hash được tính server-side.

Không lưu signals gốc — chỉ lưu fingerprint hash + encrypted signals

03

BE / FE Responsibilities

Phân công clear — BE làm phần server, FE làm phần client. Cả 2 đều có trách nhiệm riêng.

Backend (AAA Service)
  • FingerprintService — hash computation engine
  • HMAC-SHA256 — with server salt rotation
  • t_device_fingerprint — migration & model
  • 5 API endpoints — register, get, list, update, delete
  • IP /24 normalization — hỗ trợ WTS
  • AES-256-GCM encryption — signals at rest
  • Rate limiting — 10 req/min/user
  • JWT integration — gắn fingerprint_hash vào token
Frontend (App + WTS)
  • Signal collection — screen, canvas, fonts, timezone, hardware
  • 1 API call — gọi POST /deviceFingerprint sau login thành công
  • Error silent — nếu API fail, không retry, không block user
  • No UI required — chạy background, không popup/screen mới
  • WTS support — fallback nếu canvas bị chặn
  • Platform detection — iOS native vs Android vs Web
04

Hash Algorithm

Fingerprint hash là sự kết hợp của client signals + server salt + IP range, băm bằng HMAC-SHA256.

Input signals FE

  • screenResolution — Độ phân giải màn hình
  • timezone — Múi giờ (IANA)
  • language — Ngôn ngữ trình duyệt
  • canvasFingerprint — Canvas hash
  • installedFonts — Danh sách fonts (hash)
  • platform — OS + version
  • hardwareConcurrency — CPU cores

Hash computation BE

fingerprint = HMAC-SHA256(
  key     = serverSalt,
  message = signalsJSON
           + normalizedIP(/24)
           + userAgent
)
→ 64-char hex string

Configuration Database Schema

This document describes the database schema for the Configuration domain.

Overview

The Configuration domain provides system-wide configuration management including language resources, menus, FAQs, OAuth2 scopes, login methods, and service configurations. It uses TypeORM for database access.

Entity Relationship Diagram

Field Descriptions

t_admin_role

  • id: Primary key identifier for the admin role
  • role: Role name/identifier
  • description: Human-readable description of the role

t_menu_group

  • id: Primary key identifier for the menu group
  • group_name: Name of the menu group
  • description: Description of the menu group’s purpose
  • admin_role_id: Foreign key to the admin role that owns this menu group

t_menu

  • id: Primary key identifier for the menu item
  • title: Display title of the menu item
  • order: Display order/sequence of the menu item
  • icon: Icon identifier or URL for the menu item
  • is_leaf: Flag indicating if this is a leaf node (no children)
  • href: URL or route path for the menu item
  • screen_code: Code identifying the screen/view associated with this menu
  • menu_group_id: Foreign key to the menu group
  • parent_id: Foreign key to parent menu item (for hierarchical menus)

t_menu_role

  • id: Primary key identifier for the menu role
  • role_name: Name of the menu role
  • description: Description of the menu role

t_faq_group

  • id: Primary key identifier for the FAQ group
  • group_name: Name of the FAQ group/category
  • description: Description of the FAQ group

t_faq

  • id: Primary key identifier for the FAQ
  • question: The frequently asked question
  • answer: The answer to the question
  • group_id: Foreign key to the FAQ group

t_faq_review

  • id: Primary key identifier for the FAQ review
  • faq_id: Foreign key to the FAQ being reviewed
  • rating: Rating score (typically 1-5)
  • comment: Review comment/feedback
  • created_at: Timestamp when the review was created

t_lang_namespace

  • id: Primary key identifier for the language namespace
  • namespace: Namespace identifier (e.g., “common”, “errors”, “messages”)
  • description: Description of what this namespace contains

t_lang_key

  • id: Primary key identifier for the language key
  • namespace_id: Foreign key to the language namespace
  • key: The translation key identifier
  • description: Description of what this key represents

t_lang_translate

  • id: Primary key identifier for the translation
  • lang_key_id: Foreign key to the language key
  • locale: Locale code (e.g., “en”, “vi”, “ja”)
  • value: The translated text value

t_lang_resource

  • id: Primary key identifier for the language resource
  • ms_name: Microservice name that uses this resource

t_lang_resource_version

  • id: Primary key identifier for the language resource version
  • lang_resource_id: Foreign key to the language resource
  • version: Version identifier (e.g., “1.0.0”, “2.1.3”)
  • created_at: Timestamp when this version was created

t_lang_resource_file

  • id: Primary key identifier for the language resource file
  • lang_resource_version_id: Foreign key to the language resource version
  • file_name: Name of the language file
  • file_path: Path to the language file
  • created_at: Timestamp when the file was created

t_scope

  • id: Primary key identifier for the scope
  • name: Unique scope name (e.g., “read:profile”, “write:orders”)
  • uri_pattern: URI pattern that this scope grants access to
  • forward_type: Type of forwarding mechanism
  • forward_data: JSON data for forwarding configuration
  • created_by: User ID who created this record
  • updated_by: User ID who last updated this record
  • created_at: Timestamp when the record was created
  • updated_at: Timestamp when the record was last updated

t_scope_group

  • id: Primary key identifier for the scope group
  • name: Name of the scope group
  • description: Description of what the scope group represents
  • created_by: User ID who created this record
  • updated_by: User ID who last updated this record
  • created_at: Timestamp when the record was created
  • updated_at: Timestamp when the record was last updated

t_client

  • id: Primary key identifier for the OAuth2 client
  • user_id: User ID associated with this client
  • client_id: OAuth2 client identifier (public)
  • client_secret: OAuth2 client secret (private, used for authentication)
  • description: Human-readable description of the client application
  • status: Client status (active, inactive, etc.)
  • open_api_server: OpenAPI server URL
  • open_api_url: OpenAPI specification URL
  • app_version: Version of the client application
  • domain: Domain associated with the client
  • created_by: User ID who created this record
  • updated_by: User ID who last updated this record
  • created_at: Timestamp when the record was created
  • updated_at: Timestamp when the record was last updated

t_login_method

  • id: Primary key identifier for the login method
  • service_code: Code identifying the service
  • grant_type: OAuth2 grant type (authorization_code, password, etc.)
  • access_token_ttl: Access token time-to-live in seconds
  • refresh_token_ttl: Refresh token time-to-live in seconds
  • refresh_token_long_ttl: Long-lived refresh token TTL in seconds
  • multi_factor_ttl: Multi-factor authentication TTL in seconds
  • public_scopes: Comma-separated list of public scopes
  • ms_name: Microservice name
  • ms_uri: Microservice URI
  • is_default: Flag indicating if this is the default login method
  • extra_data: Additional configuration data in JSON format
  • created_by: User ID who created this record
  • updated_by: User ID who last updated this record
  • created_at: Timestamp when the record was created
  • updated_at: Timestamp when the record was last updated

t_login_method_step

  • id: Primary key identifier for the login method step
  • login_method_id: Foreign key to the parent login method
  • step_order: Order/sequence of this step in the login process
  • step_type: Type of step (password, OTP, biometric, etc.)
  • step_config: Configuration for this step in JSON format
  • created_by: User ID who created this record
  • updated_by: User ID who last updated this record
  • created_at: Timestamp when the record was created
  • updated_at: Timestamp when the record was last updated

t_data_view

  • id: Primary key identifier for the data view
  • code: Unique code identifying the data view
  • view_name: Display name of the data view
  • type: Type of data view
  • is_translated: Flag indicating if the view supports translations
  • status: Status of the data view (active, inactive, etc.)
  • filter_fields: JSON array of filter field configurations

t_service

  • id: Primary key identifier for the service
  • service_name: Display name of the microservice
  • service_code: Unique code identifying the service
  • support_phone: Support phone number
  • support_email: Support email address
  • logo_url: URL to the service logo

t_holiday

  • id: Primary key identifier for the holiday
  • holiday_name: Name of the holiday
  • holiday_date: Date of the holiday
  • description: Description of the holiday

t_interest_info

  • id: Primary key identifier for the interest info
  • key: Interest rate key/identifier
  • value: Interest rate value (as a number)

t_open_api

  • id: Primary key identifier for the OpenAPI specification
  • api_name: Name of the API
  • api_version: Version of the API specification
  • api_spec: OpenAPI specification content (JSON/YAML)
  • created_at: Timestamp when the specification was created
  • updated_at: Timestamp when the specification was last updated

t_template_resource

  • id: Primary key identifier for the template resource
  • template_name: Name of the template
  • template_type: Type of template (email, SMS, notification, etc.)
  • content: Template content (HTML, text, etc.)
  • created_at: Timestamp when the template was created
  • updated_at: Timestamp when the template was last updated

Core Entities

Menu Management

t_admin_role Defines administrative roles in the system.

t_menu_group Groups of menus organized by admin role. Each menu group belongs to an admin role.

t_menu Menu items in the system. Supports hierarchical structure (parent-child relationships) and can be assigned to multiple menu roles.

t_menu_role Roles that can access specific menus. Many-to-many relationship with menus.

Key Relationships: - AdminRole → MenuGroup (one-to-many) - MenuGroup → Menu (one-to-many) - Menu → Menu (self-referential for parent-child) - Menu ↔︎ MenuRole (many-to-many)

FAQ Management

t_faq_group Groups for organizing FAQs by category.

t_faq Frequently asked questions with answers.

t_faq_review Reviews and ratings for FAQs.

Key Relationships: - FaqGroup → Faq (one-to-many) - Faq → FaqReview (one-to-many)

Language Resource Management

t_lang_namespace Namespaces for organizing language keys (e.g., “common”, “errors”, “messages”).

t_lang_key Language keys within a namespace.

t_lang_translate Translations for language keys in different locales.

t_lang_resource Language resource files for microservices.

t_lang_resource_version Versions of language resource files.

t_lang_resource_file Individual files within a language resource version.

Key Relationships: - LangNamespace → LangKey (one-to-many) - LangKey → LangTranslate (one-to-many) - LangResource → LangResourceVersion (one-to-many) - LangResourceVersion → LangResourceFile (one-to-many)

OAuth2 & Authentication

t_scope OAuth2 scopes defining what resources clients can access.

t_scope_group Groups of scopes for easier management.

t_client OAuth2 client applications.

t_login_method Login methods configuration (password, OTP, biometric, etc.).

t_login_method_step Steps in multi-step login processes.

Key Relationships: - Scope ↔︎ ScopeGroup (many-to-many) - Client ↔︎ LoginMethod (many-to-many) - LoginMethod → LoginMethodStep (one-to-many)

System Configuration

t_data_view Data views for filtering and displaying data.

t_service Microservice configurations and contact information.

t_holiday Trading holidays and market closures.

t_interest_info Interest rate information (key-value pairs).

t_open_api OpenAPI specifications for API documentation.

t_template_resource Template resources for emails, notifications, etc.

Configuration Flow

Menu System Flow

AdminRole → MenuGroup → Menu → MenuRole
  1. AdminRole defines administrative roles
  2. MenuGroup organizes menus by role
  3. Menu items are organized hierarchically (parent-child)
  4. MenuRole controls access to menus

Language Resource Flow

Namespace → LangKey → LangTranslate
  1. LangNamespace provides organization
  2. LangKey defines translatable keys
  3. LangTranslate stores translations per locale

OAuth2 Configuration Flow

Client → LoginMethod → LoginMethodStep
Scope ↔ ScopeGroup
  1. Client applications use LoginMethod for authentication
  2. LoginMethod can have multiple LoginMethodStep for multi-factor auth
  3. Scope and ScopeGroup manage authorization

Supporting Entities

  • t_data_view: Custom data views with filters
  • t_service: Microservice registry and configuration
  • t_holiday: Calendar management for trading
  • t_interest_info: Interest rate configuration
  • t_open_api: API documentation management
  • t_template_resource: Template storage for notifications

Notes

  • All entities use TypeORM decorators (@Entity, @Column, etc.)
  • Many entities include audit fields (created_by, updated_by, created_at, updated_at)
  • Relationships are managed through TypeORM decorators (@ManyToOne, @OneToMany, @ManyToMany)
  • JSON fields are used for flexible data storage (forward_data, filter_fields)
  • The configuration service acts as a central configuration management system for the entire NHSV platform

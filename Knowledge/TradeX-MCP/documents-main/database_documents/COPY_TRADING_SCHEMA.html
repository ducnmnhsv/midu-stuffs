Copy Trading Database Schema

This document describes the database schema for the Copy Trading domain.

Overview

The Copy Trading domain manages portfolios, subscribers, trading orders, and market leader tracking. It enables users to copy trades from market leaders (brokers) to subscribers.

Entity Relationship Diagram

Field Descriptions

t_user

  • id: Primary key identifier for the user
  • login: Unique login username
  • password_hash: Hashed password
  • full_name: User’s full name
  • email: User’s email address
  • image_url: URL to user’s profile image
  • activated: Flag indicating if the account is activated
  • lang_key: Language preference key
  • activation_key: Key used for account activation
  • reset_key: Key used for password reset
  • created_date: Timestamp when the account was created
  • reset_date: Timestamp when password was reset
  • created_by: User who created this account
  • last_modified_by: User who last modified this account
  • last_modified_date: Timestamp when account was last modified
  • deactivated_at: Timestamp when account was deactivated
  • deactivated_by: User who deactivated this account

t_copy_portfolio

  • id: Primary key identifier for the portfolio
  • created_at: Timestamp when the portfolio was created
  • ml_user_id: Foreign key to the market leader user

t_copy_portfolio_details

  • id: Primary key identifier for the portfolio detail
  • portfolio_id: Foreign key to the portfolio
  • symbol: Stock symbol (e.g., “VIC”, “VNM”)
  • quantity: Number of shares
  • price: Price per share
  • order_type: Type of order (BUY, SELL, etc.)
  • created_at: Timestamp when the detail was created
  • updated_at: Timestamp when the detail was last updated

t_copy_portfolio_history

  • id: Primary key identifier for the portfolio history record
  • portfolio_id: Foreign key to the portfolio
  • action: Action performed (CREATE, UPDATE, DELETE, etc.)
  • details: Additional details about the action
  • created_at: Timestamp when the action occurred

t_copy_portfolio_detail_history

  • id: Primary key identifier for the portfolio detail history record
  • portfolio_detail_id: Foreign key to the portfolio detail
  • action: Action performed (CREATE, UPDATE, DELETE, etc.)
  • old_value: Previous value before the change
  • new_value: New value after the change
  • created_at: Timestamp when the change occurred

t_copy_subscriber

  • id: Primary key identifier for the subscriber
  • account_number: Subscriber’s account number
  • sub_number: Subscriber number/identifier
  • user_name: Username of the subscriber
  • allocated_ratio: Ratio of capital allocated for copy trading
  • order_set_type: Type of order set configuration
  • created_at: Timestamp when the subscriber was created
  • updated_at: Timestamp when the subscriber was last updated
  • device_unique_id: Unique identifier of the device
  • customer_name: Customer’s full name
  • ml_user_id: Foreign key to the market leader user being followed

t_copy_subscriber_details

  • id: Primary key identifier for the subscriber detail
  • subscriber_id: Foreign key to the subscriber
  • symbol: Stock symbol
  • quantity: Number of shares
  • price: Price per share
  • order_type: Type of order (BUY, SELL, etc.)
  • created_at: Timestamp when the detail was created
  • updated_at: Timestamp when the detail was last updated

t_copy_subscriber_history

  • id: Primary key identifier for the subscriber history record
  • subscriber_id: Foreign key to the subscriber
  • action: Action performed (SUBSCRIBE, UNSUBSCRIBE, UPDATE, etc.)
  • details: Additional details about the action
  • created_at: Timestamp when the action occurred

t_copy_trading_order

  • id: Primary key identifier for the trading order
  • portfolio_id: Foreign key to the portfolio that generated this order
  • subscriber_id: Foreign key to the subscriber receiving this order
  • symbol: Stock symbol
  • quantity: Number of shares
  • price: Price per share
  • order_type: Type of order (BUY, SELL, etc.)
  • status: Order status (PENDING, EXECUTED, CANCELLED, etc.)
  • created_at: Timestamp when the order was created
  • executed_at: Timestamp when the order was executed

t_copy_trading_register

  • id: Primary key identifier for the registration
  • user_id: Foreign key to the user registering
  • ml_user_id: Foreign key to the market leader user
  • status: Registration status (PENDING, APPROVED, REJECTED, etc.)
  • registered_at: Timestamp when registration occurred
  • created_at: Timestamp when the record was created
  • updated_at: Timestamp when the record was last updated

t_copy_market_leader_details

  • id: Primary key identifier for the market leader detail
  • user_id: Foreign key to the user who is a market leader
  • account_number: Market leader’s account number
  • total_profit_loss: Total profit/loss amount
  • total_trades: Total number of trades executed
  • win_rate: Win rate percentage
  • last_trade_date: Date of the last trade
  • created_at: Timestamp when the record was created
  • updated_at: Timestamp when the record was last updated

t_copy_market_leader_profit_loss

  • id: Primary key identifier for the profit/loss record
  • market_leader_detail_id: Foreign key to the market leader detail
  • period: Period identifier (DAILY, WEEKLY, MONTHLY, etc.)
  • profit_loss: Profit/loss amount for the period
  • trade_count: Number of trades in the period
  • win_count: Number of winning trades
  • created_at: Timestamp when the record was created

t_copy_market_leader_profit_loss_details

  • id: Primary key identifier for the profit/loss detail
  • profit_loss_id: Foreign key to the profit/loss record
  • symbol: Stock symbol
  • quantity: Number of shares
  • price: Price per share
  • profit_loss: Profit/loss amount for this symbol
  • created_at: Timestamp when the record was created

Core Entities

t_user

System users who can be either market leaders (brokers) or subscribers. This table appears in multiple repositories.

Key Relationships: - One-to-many with t_copy_portfolio (users own portfolios) - One-to-many with t_copy_subscriber (users can subscribe) - One-to-many with t_copy_trading_order (users place orders) - One-to-many with t_copy_market_leader_details (users can be market leaders)

t_copy_portfolio

Portfolios owned by market leaders. Contains the trading positions and history.

Key Relationships: - Many-to-one with t_user (owned by market leader) - One-to-many with t_copy_portfolio_details (contains positions) - One-to-many with t_copy_portfolio_history (tracks changes) - One-to-many with t_copy_trading_order (generates orders)

t_copy_portfolio_details

Individual positions within a portfolio (stocks, quantities, prices).

Key Relationships: - Many-to-one with t_copy_portfolio (belongs to portfolio) - One-to-many with t_copy_portfolio_detail_history (tracks changes)

t_copy_subscriber

Subscribers who copy trades from market leaders.

Key Relationships: - Many-to-one with t_user (subscriber user) - Many-to-one with t_user (market leader being followed) - One-to-many with t_copy_subscriber_details (has positions) - One-to-many with t_copy_subscriber_history (tracks changes) - One-to-many with t_copy_trading_order (receives orders)

t_copy_subscriber_details

Individual positions for subscribers (copied from market leader portfolios).

Key Relationships: - Many-to-one with t_copy_subscriber (belongs to subscriber)

t_copy_trading_order

Trading orders generated from market leader portfolios and executed for subscribers.

Key Relationships: - Many-to-one with t_copy_portfolio (originates from portfolio) - Many-to-one with t_copy_subscriber (executed for subscriber)

t_copy_market_leader_details

Market leader statistics and performance metrics.

Key Relationships: - Many-to-one with t_user (belongs to market leader) - One-to-many with t_copy_market_leader_profit_loss (has profit/loss records)

t_copy_market_leader_profit_loss

Profit/loss records for market leaders by time period.

Key Relationships: - Many-to-one with t_copy_market_leader_details (belongs to market leader) - One-to-many with t_copy_market_leader_profit_loss_details (has detailed records)

Copy Trading Flow

Market Leader Portfolio → Trading Order → Subscriber Portfolio
  1. Market Leader creates a portfolio (t_copy_portfolio)
  2. Portfolio contains positions (t_copy_portfolio_details)
  3. When market leader trades, orders are generated (t_copy_trading_order)
  4. Orders are distributed to Subscribers (t_copy_subscriber)
  5. Subscribers receive positions in their details (t_copy_subscriber_details)

Subscription Flow

User → Register → Subscribe → Receive Orders
  1. User registers for copy trading (t_copy_trading_register)
  2. User subscribes to a market leader (t_copy_subscriber)
  3. Subscriber receives orders from market leader portfolio
  4. Orders are executed and tracked in subscriber details

Market Leader Tracking

User → Market Leader Details → Profit/Loss → Profit/Loss Details
  1. User becomes a market leader
  2. Market leader details are tracked (t_copy_market_leader_details)
  3. Profit/loss is calculated by period (t_copy_market_leader_profit_loss)
  4. Detailed profit/loss by symbol is stored (t_copy_market_leader_profit_loss_details)

History Tracking

All major entities have history tables: - t_copy_portfolio_history - Portfolio change history - t_copy_portfolio_detail_history - Position change history - t_copy_subscriber_history - Subscriber change history

These tables track all modifications for audit and analysis purposes.

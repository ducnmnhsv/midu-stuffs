eKYC Enhancement — Store Log

Lưu trữ toàn bộ lịch sử các lần thử eKYC, tra cứu hành trình mở tài khoản, và Dashboard phân tích tỉ lệ fail theo nguyên nhân.

Giới thiệu

🔍 Vấn đề

Mỗi lần user submit lại eKYC, hệ thống xóa bản ghi PENDING cũ. Không truy vết được user fail bao nhiêu lần, fail ở bước nào.

🎯 Giải pháp

Tạo bảng ekyc_attempt_log lưu mọi lần thử (append-only). Ảnh CCCD mỗi lần fail được upload S3/MinIO.

👥 Ai hưởng lợi

Ops/CS — tra cứu case. BA/PM — phân tích Dashboard. Security — phát hiện fraud. Dev — debug production.

Vấn đề & Mục tiêu

Hiện tại — Vấn đề

  • CustomEKycService.java:211 xóa record PENDING cũ khi retry
  • App không gửi data VNPT khi fail trước khi gọi BE (blur, liveness...)
  • ekyc_ext.raw_data lưu JSON thô, không query được
  • Ảnh CCCD từng lần thử không được lưu
  • Không có Dashboard số liệu
  • default: break trong switch → cho pass qua warning không biết

Sau khi hoàn thành

  • Mọi lần thử được ghi vào ekyc_attempt_log — không bao giờ xóa
  • App gọi POST /ekycs/attempt-log ngay khi SDK fail
  • 46 cột structured — query theo fraud flags, quality score
  • Ảnh CCCD upload S3/MinIO per attempt, Admin xem được
  • Dashboard KPI 7 cards + 4 charts
  • MRZ validation + cross-check chặn giả mạo

Service Logic

EKycAttemptLogService — xử lý POST /ekycs/attempt-log

1. Parse JWT
Lấy identifierId và phoneNo từ token — App không cần truyền thủ công
2. Tính attempt_number
COUNT(*) FROM ekyc_attempt_log WHERE identifier_id = ? + 1
3. Extract VNPT fields
Nếu có vnptRawData → parse base64 JSON → map 30+ fields vào entity
4. Upload ảnh S3/MinIO
Chỉ khi có imageFrontBase64 / imageBackBase64 trong request
5. Insert vào DB
Append-only INSERT — không bao giờ UPDATE record này

CustomEKycService — cập nhật khi APPROVED

Khi Lotte reject
Repository updateOutcome() cập nhật outcome = LOTTE_REJECTED cho attempt record mới nhất
Khi account APPROVED
1. updateFinalEkycId() — link toàn bộ attempt về e_kyc.id thành công
2. Cập nhật e_kyc.total_attempts + e_kyc.first_attempt_at
Logic duplicate handling PENDING cũ giữ nguyên hoàn toàn — không break luồng hiện tại.

FE Issue — App: MRZ Validation & Cross-Check Security

Lỗ hổng hiện tại: default: break trong switch xử lý VNPT warning → bất kỳ warning code không biết đều cho user pass qua. MRZ hoàn toàn không được đọc dù SDK đã trả về.

3 Phases implement

PhaseNội dungEstimatePhụ thuộc
Phase 1 Fix default: break → block + analytics log 0.5 ngày Bắt đầu ngay
Phase 2 Đọc & validate MRZ (valid_score + cross-check 4 trường) 1 ngày Chờ VNPT confirm threshold
Phase 3 Gửi MRZ data lên BE qua POST /ekycs/attempt-log 0.5 ngày Chờ BE deploy

MRZ Cross-Check — 4 trường so sánh

🆔
Số CCCD
MRZ pos 0-8 vs OCR id
🎂
Ngày sinh
MRZ YYMMDD vs OCR DD/MM/YYYY
⚤
Giới tính
MRZ M/F vs OCR
📅
Ngày hết hạn
MRZ YYMMDD vs OCR

Nếu bất kỳ trường nào MISMATCH → block user, hiển thị cảnh báo, gọi POST /ekycs/attempt-log với outcome: MRZ_FAILED.
